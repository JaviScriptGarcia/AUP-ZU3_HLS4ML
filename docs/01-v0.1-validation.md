# v0.1-mnist-reproduction — Validation log

Date: 2026-06-09.

## What is being claimed

The MNIST CNN reference accelerator from the ICTP/AUP-ZU3-HLS4ML upstream
repository runs end-to-end on our Real Digital AUP-ZU3 board with PYNQ
3.1.1, using only `pynq.Overlay()` + AXI DMA. No VART, no XRT extensions,
no runtime patches.

This validates the entire hls4ml → Vivado → PYNQ deployment chain on our
exact hardware, confirming that the path is alive and reproducible before
we invest time in training our own models.

## How it was verified

### Test bench

- Board: Real Digital AUP-ZU3 (8 GB DDR4 variant)
- Image: official AMD / Real Digital PYNQ 3.1.1, freshly reflashed to
  guarantee a pristine state (all the patches that had been applied
  during the abandoned DPU-PYNQ exploration were rolled back by the
  reflash).
- Reference design: `02-cnn` from
  [ICTP/AUP-ZU3-HLS4ML](https://github.com/ICTP/AUP-ZU3-HLS4ML), commit
  pinned via the `upstream/ICTP-AUP-ZU3-HLS4ML` git submodule.
- Files used (verbatim from upstream):
  - `bd_wrapper_mnist.xsa` (3.4 MB) — pre-built hardware platform with
    the hls4ml-generated MNIST CNN IP wired to the PS via AXI DMA.
  - `02-cnn.ipynb` — Jupyter notebook driving inference.
  - `mnist_dataset.csv` (70 MB) — 9999-sample MNIST test set.
  - `utils.py` — Python helpers for the accelerator.

### Deployment

1. SSH copy of the four files to `/home/xilinx/jupyter_notebooks/`.
2. `mnist_dataset.csv` moved into a `test_dataset/` subfolder, as the
   notebook expects.
3. Notebook opened from Jupyter Lab at `http://192.168.1.44:9090`.
4. The path to the dataset in one cell was given as `/test_dataset/...`
   (absolute) but the dataset was actually placed relative to the
   notebook; corrected to `test_dataset/...`.
5. Cells executed in order with Shift+Enter.

**Hardware accuracy: 90.54 % on the 9999-sample test set.**

The reference software model in the upstream training notebook reaches
~99 % accuracy. The ~9 percentage point drop is the documented cost of
the quantisation-aware pruning + knowledge distillation pipeline that
makes the model small enough to be deployed fully on-chip in the
ZU3EG's BRAM + URAM budget (no DRAM traffic during inference).

## Cross-validations

- The overlay loads with `Overlay("bd_wrapper_mnist.xsa")` without
  errors, proving the `.xsa` generated by ICTP for the AUP-ZU3 is
  compatible with our PYNQ 3.1.1 runtime.
- The AXI DMA registers are populated and the `axi_dma_0` IP is visible
  in the overlay's IP dictionary.
- Inference throughput is high enough that 9999 samples complete in
  seconds with no stalls visible in the progress prints.

## Implications

1. **The hls4ml path is viable on our board.** This is the first time
   in this thesis effort that an FPGA-accelerated inference reaches a
   numeric, end-to-end result on the AUP-ZU3.
2. **The build chain in the upstream repo is directly reusable.** We
   can take any `.xsa` produced by Vivado 2024.1 + Vitis HLS 2024.1
   following ICTP's flow and run it on this same image. No image rebuild
   needed.
3. **The lessons from the DPU-PYNQ exploration carry forward as
   academic material** for the thesis chapter on critical analysis of
   FPGA AI deployment paths.
